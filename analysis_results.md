# Cloudflare AI (Whisper) 자막 추출 추출 분석 보고서

Cloudflare Workers AI (`@cf/openai/whisper-large-v3-turbo`)를 활용하여 HLS 오디오 세그먼트로부터 자막(VTT)을 추출하는 설계 및 한계점에 대한 분석 결과입니다.

## 1. 아키텍처 및 해결된 과제

### 1.1. 페이로드 제한 (5006 Error)
- **문제**: 70초 분량(7개 조각)의 오디오 데이터를 한 번에 모델로 전송할 경우, 데이터 직렬화 과정에서 Cloudflare Worker의 페이로드 크기 제한을 초과하거나 Type mismatch (`5006 Error`)가 발생.
- **해결 (30초 청크 아키텍처)**: 오디오 데이터를 Whisper 모델의 네이티브 윈도우 사이즈인 **30초(10초 세그먼트 3개)** 단위로 병합하여 전송(`[...new Uint8Array] -> Array.from`)함으로써 서버 메모리 한계와 직렬화 에러를 완벽히 우회했습니다.

### 1.2. 문맥 단절 (Context Loss)
- **문제**: 10초 단위나 30초 단위로 끊어서 전사할 경우, 문장이 중간에 잘려 문법이 어색해지는 현상 발생.
- **해결 (Iterative Prompting)**: 이전 30초 청크에서 추출된 텍스트를 다음 청크의 `initial_prompt`로 주입하여 조각 간의 문맥 연속성을 확보했습니다.

## 2. 🚨 치명적 한계점: Whisper 모델의 환각(Hallucination) 🚨

아키텍처 최적화 이후 전체 70초 분량을 안정적으로 호출하는 데는 성공했으나, **Cloudflare 서버에서 구동되는 Whisper 모델 자체의 결함**이 발견되었습니다.

1. **침묵/노이즈 구간의 환각 증세**: 영상 내에 실제 말소리가 없는 구간(예: 30초~70초)이 주입되면, 모델이 공백을 인지하지 못하고 무의미한 쓰레기 텍스트를 무한 반복해서 생성합니다.
   - *예시*: 뜬금없는 일본어(`あった!わぁ...`) 또는 힌디어(`यह लाँवाँ...`)가 수십 초간 반복됨.
2. **언어 고정(Forced Language)의 한계**: `language: "ko"` 옵션을 강제로 주입하여 일본어 환각은 억제할 수 있었으나, 여전히 빈 구간에서 힌디어 패턴 등 다른 형태의 환각이 터져 나오는 것을 확인했습니다.
3. **유지보수의 불가능성**: "특정 문자열(예: `やっぱり`, `लाँवाँ`)이 나오면 지운다"는 식의 하드코딩 필터링은 임시방편일 뿐, 모델이 언제 어떤 새로운 환각 패턴을 만들어낼지 예측할 수 없어 근본적인 해결책이 될 수 없습니다.

## 3. 최종 결론 및 권고사항

현재 **Cloudflare Workers AI 기반의 영상 자막 추출 시스템은 "빈 오디오 구간에서의 환각 제어"라는 거대한 장벽**에 부딪힌 상태입니다.

이는 코드의 버그가 아닌 **모델 파인튜닝/서빙 계층의 문제**이므로, 다음과 같은 대안을 고려해야 합니다:

1. **외부 전문 API 활용**: OpenAI의 공식 Audio API 또는 Deepgram 등 VAD(Voice Activity Detection) 처리가 완벽하게 된 상용 STT 서비스 이용.
2. **사전 VAD 처리**: Cloudflare AI에 넘기기 전에, 별도의 라이브러리를 통해 오디오에서 '말소리가 있는 구간'만 정확히 오려내어 모델에 전송하는 파이프라인 구축 (단, Workers 환경에서 무거운 미디어 처리 라이브러리 구동은 한계가 있음).
